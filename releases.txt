Release Checklist:

1. Make sure a *fresh CVS tree* builds cleanly:

   + no warnings (other than mod_auth_pam) using --enable-devel

   + few warnings/failures when running it under valgrind

   + no warnings/failures when building on the main three platforms:
     Linux, Solaris, FreeBSD
     
   + all contrib modules compile

2. Notify core in advance of pending release (24-48 hours)

3. Update NEWS

4. Update include/version.h
   Update proftpd.spec

5. Update autoconf script (if necessary)

6. Update ChangeLog and documentation
   a. Run pdd/Userguide/directives/build_output.sh and let it crank away
   b. Commit updated versions of by_name_source.sgml and
      configuration_full.html (in pdd/Userguide/directives/output/)
      as proftpd/docs/Configuration.{sgml,html}
   c. Run pdd/FAQ/build_proftpd_faq and let it crank away
   d. Commit updated versions of faq_full.html as proftpd/docs/faq.html

7. Tag CVS module for release.
  a. cvs rtag <tag> proftpd, e.g. R1_2_5RC3

8. Create tarball (.tar.gz, .tar.bz2)
  a. cvs checkout -r <tag>
  b. remove CVS subdirs, .cvsignore files before tar'ing up.
     ./configure
     make dist

9. Sign/hash tarballs (.tar.{bz2,gz}.asc (PGP), .tar.{bz2,gz}.md5 (MD5))
  a. sign tarballs, e.g. pgp -sba
  b. PGP: write email announcement to file, use pgp -sta <file> to
     produce signed message
  c. PGP: verify a test email of the announcement to self (pgp (no options))

10. Update the web site:
   * Update md5_pgp.epl file w/ new MD5 sum, PGP signatures.
   * Update include/globals.epl with new version number(s).
   * Update index.epl to announce the new release.
   * add docs/NEWS-versionnum

11. Upload tarball, MD5 signature, PGP signatures, RPMs
   a. Upload files to core account (currently boost.horde.net, user proftpd),
      in the proper place under ~proftppd/ftp.proftpd.org/.  Make sure to
      update MD5SUMS file in distrib/packages/RPMS.
   c. Run "~/.bin/update-all.sh" to propagate the new files as necessary.

12. Announce on lists: -announce, -devel, -users

13. Update Bugzilla with new component version tag

14. Post release to freshmeat, c.o.l.a

15. Update include/version.h in CVS.
